# TeleBid

Telegram Mini App и бот для прямых и обратных аукционов рекламы в Telegram-каналах. Проект одновременно является рабочим MVP и экспериментальным стендом НИР по архитектуре конкурентных веб-торгов.

## Что умеет MVP

- владелец канала выставляет свободный рекламный слот, а рекламодатели повышают ставки;
- рекламодатель публикует тендер, а владельцы каналов снижают цену размещения;
- карточка слота публикуется в Telegram-канале и ведёт участника в Mini App;
- ставки и состояние торгов обновляются через WebSocket;
- после reconnect клиент получает пропущенные версии событий;
- перебитая ставка, победа и завершение торгов сохраняются в серверном inbox, восстанавливаются Mini App по cursor и доставляются Telegram-ботом с retry;
- антиснайпер продлевает торги при ставке перед завершением;
- `commandId` защищает от повторного применения одной ставки.

## Исследовательская задача

Telegram Mini App может быть свёрнут или закрыт в момент торгов. При этом WebSocket обрывается, клиент пропускает обновления, а несколько одновременных ставок могут конкурировать за одно состояние на сервере. Простая рассылка текущей цены не гарантирует ни правильного победителя, ни актуальный экран после reconnect, ни получение результата вне Mini App.

Сетевой пилот проверяет пять независимых предположений: транзакционная обработка и идемпотентность сохраняют корректный результат торгов; SSE и WebSocket уменьшают live-задержку относительно polling; версионированный replay восстанавливает клиент после разрыва; hybrid recovery выбирает меньший корректный payload; durable inbox Mini App и Telegram Bot fallback доставляют итог торгов после закрытия интерфейса и временного отказа Bot API.

Детерминированное сравнение базовой и предлагаемой архитектур доступно как отдельная демонстрационная модель. Его результаты не смешиваются с сетевым датасетом пилотной серии.

## Запуск в Telegram

Нужны Docker Desktop, Node.js 22+ и токен бота от `@BotFather`.

```bash
npm install
npm run telegram
```

При первом запуске появится `.env`. Вставьте в него только токен:

```env
TELEGRAM_BOT_TOKEN=1234567890:ваш_токен
```

Затем повторите:

```bash
npm run telegram
```

Команда сама:

1. поднимет PostgreSQL, API и веб-интерфейс;
2. использует локальный ngrok-туннель, если он запущен, либо создаст временный HTTPS-адрес через Cloudflare Tunnel с резервным переключением на runlocal;
3. установит этот адрес в кнопку меню бота;
4. настроит команды `/start`, `/app` и `/help`;
5. выведет и откроет ссылку на бота.

Для публикации слота добавьте бота администратором публичного канала с правом отправки сообщений. Тот же Telegram-пользователь должен быть владельцем или администратором канала.

Быстрые туннели предназначены для разработки. Фоновый наблюдатель отслеживает адрес ngrok или runlocal и автоматически обновляет кнопку Mini App в Telegram. Для постоянного размещения укажите стабильный HTTPS-адрес в `MINI_APP_URL` и разверните сервис на сервере.

## Локальный demo-режим

Токен Telegram не нужен:

```bash
npm install
npm run lab
```

Открыть:

- Mini App: `http://localhost:4173`;
- архитектурный эксперимент: `http://localhost:4173/?mode=lab`;
- Swagger API: `http://localhost:8080/api`.

В demo-режиме можно переключаться между рекламодателем и владельцем канала, создавать оба типа торгов и делать ставки.

## Экспериментальный стенд НИР-2

Исследовательский контур запускается отдельно от Telegram-бота: у него собственные API, PostgreSQL, Toxiproxy, порты и Docker volume. Живой Mini App продолжает работать и не получает тестовые аукционы.

Короткая проверка стенда:

```bash
npm run research:up
npm run research:wait
npm run research:quick
```

Полная пилотная серия одной командой:

```bash
npm run research
```

Она сравнивает polling, SSE и WebSocket для прямых и обратных торгов, изменяет количество клиентов и интенсивность ставок, вносит latency/jitter, физически разрывает соединения через Toxiproxy и повторяет уже принятые `commandId`. В reconnect-сценарии аукцион завершается при отключённом Mini App, а адаптер Telegram отклоняет первую отправку каждого сообщения: стенд проверяет replay уведомлений и успешный retry.

Каждый запуск сохраняется в отдельной папке `datasets/<run-id>/`: сырые события, команды, доставки уведомлений Mini App, клиентские и trial-метрики, агрегаты с bootstrap 95% интервалом, JSON-манифест и готовый Markdown-отчёт. Последний путь записывается в `datasets/latest.json`.

Остановить или полностью очистить только исследовательский контур:

```bash
npm run research:down
npm run research:reset
```

Детерминированная экранная модель двух архитектур остаётся доступна отдельно:

```bash
npm run research:model
```

Полная методика описана в `docs/research-design.md`, схема данных — в `docs/dataset-schema.md`.

## Архитектура

- `apps/web` — React + TypeScript Telegram Mini App и экран эксперимента;
- `apps/api/src/auction` — транзакционная обработка прямых и обратных торгов;
- `apps/api/src/sync` — WebSocket, версии событий и восстановление клиента;
- `apps/api/src/telegram` — проверка `initData`, бот, публикация в канал и доставка уведомлений;
- `apps/api/src/experiment` — воспроизводимая модель двух архитектур;
- `tools/load` — реальные polling/SSE/WebSocket-прогоны, статистика и экспорт датасета;
- `docs/research-design.md` — полная постановка исследования и методика.
